
# خطة مشروع المنصة التعليمية التفاعلية (Static SPA)

## 1. نظرة عامة على المشروع (Project Overview)
منصة تعليمية بتصميم تطبيق الصفحة الواحدة (Single Page Application). تعتمد بالكامل على تقنيات الويب الأساسية بدون إطارات عمل لتوفير أقصى سرعة وأداء. المنصة تدعم عرض الفيديوهات، ملفات PDF، اختبارات تفاعلية، ألغاز يومية، وتحتوي على آلة حاسبة علمية مدمجة.

**التقنيات المستخدمة:** HTML5, CSS3, Vanilla JavaScript, JSON.

---

## 2. تخطيط واجهة المستخدم (UI/UX Layout)

### نسخة الحاسوب (Desktop Layout)
يتم تقسيم الشاشة إلى 3 أعمدة رئيسية باستخدام `CSS Grid` أو `Flexbox`:
- **العمود الأيمن (القائمة الجانبية - Navigation):** يحتوي على شجرة المواد والدروس (بنظام القوائم المنسدلة Accordion).
- **المنتصف (منطقة العرض - Content Area):** مساحة ديناميكية تتغير بناءً على اختيار المستخدم (يتم فيها تضمين الـ iframes للفيديوهات والـ PDFs، أو توليد واجهة الاختبارات).
- **العمود الأيسر (الأدوات - Tools):** يحتوي على الآلة الحاسبة العلمية الثابتة.

### نسخة الهاتف (Mobile Layout)
- إخفاء الآلة الحاسبة العلمية كلياً (لتوفير المساحة).
- تحويل العمود الأيمن إلى قائمة جانبية مخفية (Hamburger Menu / Off-canvas drawer) تظهر عند النقر على زر أعلى الشاشة.
- منطقة العرض تأخذ عرض الشاشة بالكامل (100%).

---

## 3. هيكلة البيانات (Data Architecture)

يتم تخزين المحتوى بالكامل في ملفات JSON لتسهيل الصيانة والتحضير المستقبلي للذكاء الاصطناعي.

### أ. ملف المناهج (`curriculum.json`)
هيكل الشجرة (Tree Structure) سيكون كالتالي:
1. **قائمة الصفوف** (مثال: الصف الأول الثانوي).
   - **المواد** (مثال: فيزياء).
     - **رابط كتاب المادة** (PDF عام للمادة بالكامل).
     - **الفصول** (الفصل الأول / الفصل الثاني).
       - **الوحدات**
         - **الدروس**
           - رابط الدرس الرقمي (يوتيوب).
           - رابط الملخص (PDF).
           - رابط أسئلة سنوات سابقة (PDF أو صور).
           - قائمة بأسئلة "الاختبار السريع" (مصفوفة من الأسئلة، الخيارات، والإجابة الصحيحة).

### ب. ملف الألغاز (`puzzles.json`)
مصفوفة تحتوي على كائنات (Objects)، كل كائن يمثل لغزاً مستقلاً:
- تاريخ اللغز (Date - للتحقق منه برمجياً).
- نص سؤال اللغز.
- الإجابة الصحيحة.
- شرح الإجابة (لعرضه في اليوم التالي).

## 4. هيكلة الملفات (File Structure)

تم تقسيم الملفات بشكل (Modular) لسهولة التوثيق والتطوير المستقبلي:

```text
project-root/
│
├── index.html               # الهيكل الأساسي للصفحة (الأعمدة الثلاثة فارغة من المحتوى)
│
├── css/
│   ├── main.css             # المتغيرات الأساسية (Colors, Fonts) والإعدادات العامة
│   ├── layout.css           # تقسيم الشاشة (Grid/Flexbox) للأعمدة الثلاثة
│   ├── components.css       # تنسيقات الأزرار، القوائم المنسدلة، والاختبارات
│   ├── calculator.css       # تنسيق الآلة الحاسبة العلمية
│   └── responsive.css       # (Media Queries) لتصميم الهاتف المحمول
│
├── js/
│   ├── app.js               # الملف المركزي الذي يربط باقي الملفات ويدير تهيئة الموقع
│   ├── dataService.js       # دالة الجلب (Fetch) لملفات الـ JSON
│   ├── uiController.js      # التعامل مع الـ DOM (فتح القوائم، عرض المحتوى في المنتصف)
│   ├── calculator.js        # المنطق الرياضي للآلة الحاسبة العلمية
│   ├── quizEngine.js        # توليد الاختبارات السريعة، حساب النتيجة وعرضها
│   └── dailyPuzzle.js       # منطق مقارنة التواريخ وعرض لغز اليوم وإجابة الأمس
│
└── data/
    ├── curriculum.json      # داتابيز المناهج
    └── puzzles.json         # داتابيز الألغاز
```

## 5. المنطق البرمجي والميزات الأساسية (Core Logic)

1. **الآلة الحاسبة العلمية (`calculator.js`):**
   - تعمل بشكل مستقل تماماً عن باقي الموقع.
   - تستقبل المدخلات عبر أزرار الواجهة وتستخدم دوال الجافاسكربت الرياضية `Math.` لحساب العمليات المعقدة.

2. **لغز اليوم (`dailyPuzzle.js`):**
   - عند تحميل الموقع، تقرأ الجافاسكربت تاريخ اليوم من نظام المستخدم `new Date()`.
   - تبحث في ملف `puzzles.json` عن اللغز المطابق لتاريخ اليوم وتعرضه.
   - تبحث عن لغز يوم أمس وتعرض إجابته أسفل اللغز الجديد.

3. **الاختبار السريع (`quizEngine.js`):**
   - دالة تستقبل البيانات من الـ JSON.
   - تقوم بمسح محتوى "المنتصف" وتوليد عناصر HTML (Radio buttons) للأسئلة ديناميكياً.
   - عند النقر على "تحقق"، تقارن إجابات المستخدم بالبيانات وتعرض النتيجة (مثال: 4/5).

4. **توجيه الذكاء الاصطناعي المستقبلي (AI Future-Proofing):**
   - تم تصميم `quizEngine.js` و `dailyPuzzle.js` لكي تعرض البيانات القادمة إليها كـ Parameter.
   - في المستقبل، سيتم استبدال دالة القراءة من (ملف JSON محلي) بدالة ترسل طلب (Fetch API) للخادم الذي يحتوي على الـ AI، ليقوم بتوليد الأسئلة وإرجاعها بنفس صيغة الـ JSON الحالية، ولن يتطلب ذلك أي تغيير في واجهة المستخدم (UI).

## 6. مراحل التنفيذ (Development Roadmap)

- [ ] **المرحلة الأولى:** بناء الهيكل الأساسي (HTML) والتنسيق العام للأعمدة (CSS Layout).
- [ ] **المرحلة الثانية:** تصميم الآلة الحاسبة (HTML/CSS) وكتابة المنطق الخاص بها (JS).
- [ ] **المرحلة الثالثة:** إنشاء ملفات الـ JSON (بيانات وهمية للتجربة).
- [ ] **المرحلة الرابعة:** كتابة كود الجافاسكربت الخاص بجلب الـ JSON وبناء القائمة الجانبية (الشجرة) ديناميكياً.
- [ ] **المرحلة الخامسة:** ربط أزرار القائمة الجانبية بمنطقة المنتصف (عرض الـ iframe للـ PDF أو اليوتيوب).
- [ ] **المرحلة السادسة:** برمجة محرك الاختبارات السريعة ولغز اليوم.
- [ ] **المرحلة السابعة:** تحسين التصميم للهواتف الذكية (Mobile Responsiveness).
- [ ] **المرحلة الثامنة:** إضافة التوثيق البرمجي (Comments) داخل كل ملف.
